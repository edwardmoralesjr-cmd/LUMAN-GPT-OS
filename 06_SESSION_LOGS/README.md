# LUMAN OS Session Logs

This folder stores durable summaries of meaningful LUMAN OS sessions.

## Purpose

Session logs preserve the important decisions, updates, tasks, and open loops that happen during Edward’s work with LUMAN OS.

Use session logs when a conversation creates durable value, such as:

- Core LUMAN OS changes
- New or updated commands
- Major project decisions
- Roseborn or Lucid Syntax canon updates
- Life OS or OMNI-Vault system changes
- GitHub maintenance work
- Important summaries that should not stay buried in chat history

## Folder Structure

Use year-month folders:

- `2026-06/`
- `2026-07/`
- `2026-08/`

Use this filename format:

`YYYY-MM-DD-short-session-title.md`

Example:

`2026-06-17-luman-github-updater-cleanup.md`

## Standard Session Log Template

Use this structure:

# Session Title

Date: YYYY-MM-DD  
Main Vault: [vault name]  
Secondary Vaults: [vaults]

- `path/file.md`

## Notes for Future LUMAN

Durable context that will help the next session continue cleanly.

## Rules

- Keep logs concise but complete.
- Do not store secrets, credentials, seed phrases, live financial balances, employer-confidential data, or sensitive private records.
- Use logs for durable system memory, not every casual exchange.
- When a summary packet is committed, create or update the matching session log.
