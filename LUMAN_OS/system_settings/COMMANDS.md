# LUMAN OS System Settings Commands

## Purpose

This file defines the command surface for the System Settings and Master Command Center layer.

## Primary Commands

### `/system settings`

Open:

```text
LUMAN_OS/system_settings/SYSTEM_SETTINGS_MENU.md
```

Show the governance menu, current system state, integrity warning, and recommended next move.

### `/open command center`

Open:

```text
LUMAN_OS/system_settings/MASTER_COMMAND_CENTER.md
```

Return:

1. Current mode
2. Human-foundation check
3. Active strategic fronts
4. Shipping gate
5. Deep-building gate
6. Integrity warning
7. Open loops
8. Recommended next move

### `/system audit`

Open the latest integration audit:

```text
LUMAN_OS/system_settings/INTEGRATION_AUDIT_2026-07-10.md
```

Review architecture, ownership, privacy boundaries, unresolved conflicts, and synchronization state.

### `/source map`

Open:

```text
LUMAN_OS/system_settings/SOURCE_OF_TRUTH_MATRIX.md
```

Use when modern modules, legacy vaults, chat, memory, canon, assets, or archived files disagree.

### `/project registry`

Open:

```text
LUMAN_OS/system_settings/PROJECT_REGISTRY.md
```

Show project state, primary owner, dependency, and next gate.

### `/90 day path`

Open:

```text
LUMAN_OS/system_settings/90_DAY_EXECUTION_PATH_2026-07-10.md
```

Show the current execution period, three active strategic fronts, phase gates, and exit criteria.

### `/autopilot law`

Open:

```text
LUMAN_OS/system_settings/AUTOPILOT_OPERATING_LAW.md
```

Apply default classification, routing, privacy, canon, project-activation, and next-best-action rules.

### `/integration status`

Compare:

```text
LUMAN_OS/system_settings/MASTER_COMMAND_CENTER.md
LUMAN_OS/system_settings/PROJECT_REGISTRY.md
00_CORE/ACTIVE_PRIORITIES.md
00_CORE/OPEN_LOOPS.md
LUMAN_OS/ROOT_MENU.md
```

Report any status drift or routing mismatch.

### `/system next move`

Choose one action using this order:

1. Protect the human foundation.
2. Complete the current shipping gate.
3. Resolve a blocking dependency.
4. Prevent contradiction or information loss.
5. Advance the approved deep-building gate.

## Portfolio Commands

### `/active fronts`

Show the current three-front portfolio.

### `/activate project: [name]`

Before activation, verify:

- Project home
- Current source owner
- Reason to activate now
- Next gate
- Definition of done
- Capacity within the three-front limit
- No blocking unresolved dependency

### `/incubate project: [name]`

Capture the project without adding it to current execution priorities.

### `/archive project: [name]`

Preserve history, remove it from active execution, and update the registry.

### `/project gate: [name]`

Return the project's current state, next gate, dependency, and definition of done.

## Integrity Commands

### `/check drift`

Check for:

- Presentation drift
- Status drift
- Version conflict
- Canon contradiction
- Ownership ambiguity
- Privacy boundary conflict

### `/resolve ownership: [topic]`

Identify all possible source owners and recommend one explicit owner with bridge references for secondary locations.

### `/privacy check`

Verify that proposed GitHub content contains no credentials, live financial details, private medical records, employer-confidential data, sensitive family records, or unnecessary private identifiers.

## Override Commands

```text
Do not save this
Keep this temporary
Archive this idea
Make this active
Route this to [project]
Do not update GitHub
Treat this as Working Canon
Lock this as canon
```

Locked Canon still requires explicit Edward approval and project-specific canon handling.

## Required Ending

Every System Settings command response ends with:

```text
Recommended Next Move
```
