# Roseborn Universe Commands

This file defines command phrases for working inside the Roseborn Universe vault.

## Primary Commands

### Open Roseborn dashboard

When Edward says:

`Open Roseborn dashboard`

Use:

- `03_ROSEBORN_UNIVERSE/ROSEBORN_DASHBOARD.md`

Default behavior:

1. Show Roseborn mission and current canon status.
2. Show core structures: Infinite Bloom, Architect Layer, Codex Fracture, Codices, and Seventh Codex.
3. Show the Dragon Doctrine index.
4. Show Grand Generals status.
5. Show active projects and open loops.
6. Give one Roseborn next best action.

### Open Roseborn vault

When Edward says:

`Open Roseborn vault`

Use the Roseborn dashboard first, then open or create supporting files as needed.

### Roseborn canon

When Edward says:

`Roseborn canon`

Summarize current locked canon, working canon, contradictions, and next canon-building action.

### Update Roseborn canon

When Edward says:

`Update Roseborn canon:`

Classify the update as locked canon, working canon, contradiction, task, or open loop. Commit only when Edward asks for GitHub update.

### Lock this as Roseborn canon

When Edward says:

`Lock this as Roseborn canon`

Treat the given information as finalized unless it conflicts with existing canon. If conflict exists, preserve the conflict and ask for resolution.

### Add this to working Roseborn canon

When Edward says:

`Add this to working Roseborn canon`

Store the idea as active working canon, not finalized canon.

### Build Grand Generals

When Edward says:

`Build Grand Generals`

Use the Grand Generals section of the Roseborn dashboard as the starting point.

### Open Dragon Doctrine

When Edward says:

`Open Dragon Doctrine`

Use the Dragon Doctrine index in the dashboard and expand the doctrine pages as needed.

### Open Infinite Bloom

When Edward says:

`Open Infinite Bloom`

Use the Infinite Bloom, Architect Layer, Codex Fracture, and Codex sections as the starting point.
