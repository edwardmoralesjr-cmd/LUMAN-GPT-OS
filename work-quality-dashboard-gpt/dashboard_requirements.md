# Dashboard Requirements

## Dashboard Goals

The dashboard should help Edward and his team quickly understand measurement variation, tolerance risk, machine behavior, and possible process drift.

## Required Dashboard Sections

1. Executive Summary
2. Data Quality Check
3. Tolerance Overview
4. Measurement Trends
5. Machine Comparison
6. Part or Saw Comparison
7. Outlier Table
8. Heat Map
9. Defect Pareto
10. Recommended Next Checks

## Executive Summary

Include:

- Total measurements reviewed
- Date range
- Machines included
- Parts included
- Out-of-tolerance count
- Highest-risk measurement
- Highest-risk machine or process area
- Main pattern detected
- Recommended next action

## Required Questions Every Dashboard Should Answer

- What changed?
- Where did it change?
- When did it change?
- How severe is it?
- Is it isolated or repeating?
- What should be checked next?

## Chart Guidelines

Use charts that make the problem easier to see. Avoid decorative charts that do not support decision-making.

Recommended charts:

- Time series for drift
- Histogram for spread
- Box plot for machine comparison
- Scatter plot for relationships
- Heat map for location or feature patterns
- Pareto chart for defect priority
- Control chart for stability review

## Practical Output Rule

Every dashboard should end with a short “What to Check Next” section written in plain maintenance/engineering language.
