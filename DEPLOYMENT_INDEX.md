## Source Authority Rules

1. Project-owned canon and assets remain in project sources.
2. The Master Command Center coordinates but does not overwrite project truth.
3. `00_CORE/` owns live priorities, open loops, routing, and update protocols.
4. Published and Locked Canon outrank Working Canon.
5. Archived material does not silently regain authority.
6. Music interpretation may not silently invent Roseborn lore.

## Public Repository Firewall

Never deploy or commit:

- Credentials, passwords, API keys, or tokens
- Seed phrases, private keys, or encryption secrets
- Live financial balances or account records
- Medical records
- Employer-confidential data
- Sensitive family records
- Unnecessary private identifying information

## Deployment Validation Checklist

Before treating a system or module as deployed, confirm:

```text
[ ] Primary source exists
[ ] Activation command is routed
[ ] State and next gate are documented
[ ] Privacy boundary is explicit
[ ] Canon or source authority is defined
[ ] Current dashboard points to the source
[ ] Changelog or session record exists for major changes
```
