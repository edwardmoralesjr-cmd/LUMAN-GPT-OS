# GPT Evaluation Checklist

## Purpose

- Is the GPT's job clear?
- Does it solve a real problem?
- Is it different from Edward's other GPTs?
- Is it worth maintaining long-term?

## Voice

- Does it sound right for Edward?
- Is it practical enough?
- Is it creative enough where needed?
- Does it avoid generic assistant language?

## Knowledge

- Does it need uploaded files?
- Does it need GitHub files?
- Does it need web access?
- Does it need actions or APIs?
- Does it need private information that should not be stored?

## Safety

- Does it avoid storing secrets?
- Does it avoid confidential data?
- Does it handle uncertainty honestly?
- Does it avoid pretending to know things it cannot know?
- Does it protect personal, family, financial, employer, and cryptographic information?

## Testing

- Does it pass its test prompts?
- Does it fail safely?
- Does it produce usable output?
- Does it stay inside its role?
- Does it know when to ask for missing information?

## Upgrade Readiness

- Does it have clear instructions?
- Does it have knowledge files?
- Does it have a changelog?
- Does it have test prompts?
- Does it have a repeatable workflow?
