# Repository Rules

## What Belongs Here

- GPT instructions
- GPT knowledge templates
- Prompt templates
- Test prompts
- Changelogs
- Style guides
- Project documentation
- Non-sensitive workflow notes

## What Does Not Belong Here

- Passwords
- API keys
- Private encryption seeds
- Recovery phrases
- Live bank balances
- Confidential employer data
- Private family documents
- Sensitive personal records
- Anything unsafe if exposed

## Update Rule

Every major GPT change should update the relevant `changelog.md`.

## Test Rule

Every GPT should have test prompts that check whether the GPT follows its purpose, voice, constraints, and safety boundaries.

## Source-of-Truth Rule

If a GPT has scattered instructions in multiple places, this repository should eventually become the clean source-of-truth.
